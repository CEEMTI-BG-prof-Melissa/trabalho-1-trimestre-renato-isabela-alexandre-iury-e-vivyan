Requisitos Funcionais Requisitos não funcionais Requisitos Inverso

Requisitos Funcionais

  • O sistema deve prover um cadastro de gerenciamento do acervo de livros
  • O sistema deve prover a visualização de livros disponíveis/Indisponíveis
  • O sistema deve prover a reserva de livros antecipadamente
  • O sistema deve prover o controle de empréstimos e devoluções
  • O sistema deve prover o envio de lembretes automáticos
  • O sistema deve prover o acesso a livros digitais
  • O sistema deve prover a leitura online (celular/tablet)
  • O sistema deve prover a recomendações de livros personalizadas
  • O sistema deve prover a sugestões de leitura por professores
  • O sistema deve prover os relatórios e estatísticas

Requisitos não Funcionais

  • O sistema deve ser fácil de usar, garantindo boa usabilidade para os usuários.
  • O sistema deve ser responsivo, permitindo acesso adequado em celulares e tablets.
  • O sistema deve apresentar bom desempenho, com respostas rápidas às ações dos usuários.
  • O sistema deve garantir segurança por meio de login e autenticação confiáveis.
  • O sistema deve assegurar a privacidade dos dados dos usuários.
  • O sistema deve possuir alta disponibilidade, permitindo acesso a qualquer momento.
  • O sistema deve ter uma interface intuitiva e organizada.
  • O sistema deve ser escalável, suportando um grande número de usuários sem perda de desempenho.

Requisitos Inverso

  • O sistema não deve permitir acesso de usuários não autorizados.
  • O sistema não deve permitir acesso às funcionalidades sem autenticação.
  • O sistema não deve permitir o armazenamento de dados incorretos.
  • O sistema não deve apresentar lentidão ou instabilidade durante o uso.
  • O sistema não deve permitir a perda de dados dos usuários.
