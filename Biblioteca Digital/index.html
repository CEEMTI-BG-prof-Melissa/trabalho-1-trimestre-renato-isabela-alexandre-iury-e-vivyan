Requisitos Funcionais Requisitos não funcionais Requisitos Inverso

Requisitos Funcionais

  • Cadastro de Gerenciamento do acervo de livros
  • Visualização de livros disponiveis/Inidsponiveis
  • Reserva de elivros antecipadamente
  • Controle de Empréstimos e devoluções
  • Envio de lembretes automáticos
  • Acesso a livros digitais
  • Leitura online (celular/tablet)
  • Recomendações de livros personalizadas
  • Sugestões de leitura por professores
  • Relatorios e estatisticas

Requisitos não Funcionais

  • Sistema deve ser facil de usar (usabilidade)
  • Acesso em celulares e tablets (responsivo)
  • Bom desempenho (respostas rapidas)
  • Segurança com login e autenticação
  • Privacidade dos dados dos usuarios
  • Alta disponibilidade (Acesso a qualquer hora)
  • Interface intuitiva e organizada
  • Escalabilidade (suportar muitos usuarios)

Requisitos Inverso

  • Não permitir acesso de pessoas externas
  • Não permitir uso de sistemas sem login
  • Não permitir informações incorretas sem login
  • Não permitir que o sistema seja lento ou instavel
  • Não permitir perda de dados dos usuarios
